SQL Database Management System

This project involves designing and managing a relational database using SQL. It includes creating tables, inserting data, and performing queries.

Tool Used: MySQL Workbench

In this project, I designed and implemented a database schema to represent my CV structure. The purpose was to practice SQL skills such as creating tables, inserting data, querying information, and structuring real-world information into a database.

Key Features of the Project:

  • Table Creation:
    • Designed and created multiple tables for core CV sections, including:
      • Personal_Details: Capturing fields like name, email, phone number, address, and LinkedIn profile.
      • Education: Storing information such as institution name, degree, field of study, start and end dates, and achievements.
      • Work_Experience: Documenting past roles with fields for company name, job title, start and end dates, and responsibilities.
      • Skills: Highlighting technical, soft, and language skills.
      • Certifications: Capturing certifications with fields like certification name, issuing authority, and issue date.
      • Projects: Detailing key projects with fields for project name, description, technologies used, and the year completed.
  • Data Insertion:
    • Populated the tables with realistic sample data to simulate a professional CV.
  • Queries and Data Analysis:
    • Wrote and executed SQL queries to retrieve insights, such as:
      • Listing education details sorted by recent dates.
      • Filtering work experiences based on a specific role or industry.
      • Searching for a particular skill or technology used in projects.
      • Generating a summary of certifications and when they were issued.
  • Normalization:
    • Ensured the database design adhered to normalization principles by avoiding data redundancy and maintaining relationships through primary and foreign keys.
  • Practice with Joins and Aggregations:
    • Practiced advanced SQL techniques such as JOINs to combine data from multiple tables (e.g., linking projects to technologies or skills used).
    • Used aggregate functions to summarize data (e.g., counting the number of certifications or projects).
  • Output:
    • Created a comprehensive report detailing the SQL operations used and the final structure of the CV database.

Python Quiz Project

Language Used: Python

This project involved creating an interactive quiz application using Python. The aim was to develop a program that tests the user's knowledge on a variety of topics, allowing for practice with Python programming concepts such as conditional statements, loops, functions, and data structures.

Key Features of the Project:

  • Dynamic Question Handling:
    • Implemented a quiz with multiple questions and varying difficulty levels.
    • Questions were stored in a structured format, such as a dictionary or JSON file, for easy addition or modification.
  • User Interaction:
    • Designed the application to interact with the user through the console or terminal.
    • Allowed users to input answers and receive immediate feedback on whether their response was correct or incorrect.
  • Scoring System:
    • Incorporated a scoring mechanism to track the user's performance.
    • Displayed a final score summary at the end of the quiz, showing the number of correct and incorrect answers.
  • Code Optimization:
    • Utilized functions for code modularity, such as functions for displaying questions, checking answers, and calculating scores.
    • Used loops and conditionals to handle user inputs and ensure running the program without errors.

Outcome:

The project provided hands-on experience with Python programming, especially focusing on user interaction. The final application was functional, engaging, and easily expandable for future enhancements.
